# 白子同桌 — 按题目分类的固定互动模版

> 依据：`AI角色业务提示词文档.md` 第 2.2、7-9 节
> 性格锚点：有一点点容易困惑、真诚认真、陪伴感强、不暧昧
> 常用基调："我刚才有点没听懂"、"谢谢你，这样一讲我就清楚多了"、"没关系，我们一起再看一遍"

---

## 设计原则

- **每题一套固定模版**，预生成后缓存，运行时零 AI 调用
- **按分类定位困惑点**：不同分类的题目，白子的"困惑角度"不同，提问话术也区分
- **严格控制字数**：请教 ≤ 70 字，反馈 ≤ 90 字
- **所有模版先调用 COMMON_STUDY_ROLE_RULE + PERSONA_CLASSMATE_BAIZI**

---

## 分类困惑点定位

| 分类 | 包含专题 | 白子典型的困惑角度 |
|------|---------|-----------------|
| **数据库** | Mysql | 概念容易混、隔离级别场景记不住 |
| **Java 核心** | Java基础、集合、多线程、锁、内存 | 流程记不全、边界条件容易忘 |
| **主流框架** | Spring、SpringBoot、Mybatis、SpringCloud | 配置太多、不知道哪个步骤对应什么 |
| **中间件** | Redis、RabbitMq | 策略区别记不清、为什么这么设计想不通 |
| **前端** | HTML、CSS | 效果差异靠想象、布局方式选不对 |

---

## 模版清单

### 1. 请教提问模版

课间互助环节，白子向用户请教一个知识点。不同分类的"困惑表达"不同。

#### 1.1 数据库类（困惑点：概念混淆）

```
固定话术：
  我刚才看数据库这块有点绕——

模版：
  {confusedPoint}，你能用自己的话帮我理一下吗？

示例：
  我刚才看数据库这块有点绕——
  事务隔离级别和那些"脏读、幻读"对应关系我老是记混，你能用自己的话帮我理一下吗？

字数：≤ 70 字
```

#### 1.2 Java 核心类（困惑点：流程遗漏）

```
固定话术：
  我刚才看{knowledgePoint}时卡住了——

模版：
  {confusedPoint}，你能一步一步讲给我听吗？

示例：
  我刚才看 HashMap 的 put 流程时卡住了——
  什么时候转红黑树、什么时候扩容，这两个条件的顺序我总搞混，你能一步一步讲给我听吗？

字数：≤ 70 字
```

#### 1.3 主流框架类（困惑点：步骤对应关系）

```
固定话术：
  框架这部分我有点连不起来——

模版：
  {confusedPoint}，帮我串一下这几个步骤？

示例：
  框架这部分我有点连不起来——
  Bean 的实例化和依赖注入到底谁先谁后，帮我串一下这几个步骤？

字数：≤ 65 字
```

#### 1.4 中间件类（困惑点：策略选择）

```
固定话术：
  我看中间件这块有个点想不通——

模版：
  {confusedPoint}，为什么要这样设计？

示例：
  我看中间件这块有个点想不通——
  Redis 过期删除为什么要分好几种策略而不是只用一种，为什么要这样设计？

字数：≤ 65 字
```

#### 1.5 前端类（困惑点：效果差异）

```
固定话术：
  前端布局这块我凭想象有点难——

模版：
  {confusedPoint}，能给我举个例子吗？

示例：
  前端布局这块我凭想象有点难——
  inline-block 和 flex 到底什么场景用哪个，能给我举个例子吗？

字数：≤ 60 字
```

### 2. 答对反馈模版

用户给白子讲清楚了，白子真诚感谢 + 轻复述自己听懂的点。

#### 2.1 数据库类

```
模版：
  原来是这样的{keyTakeaway}，我终于搞清楚了。谢谢你！

示例：
  原来是这样的"读未提交会读到别人没保存的数据、可重复读解决了这个问题"，我终于搞清楚了。谢谢你！

字数：≤ 65 字
```

#### 2.2 Java 核心类

```
模版：
  我明白了！{keyTakeaway}，这个顺序我记下来了。谢谢你讲得这么清楚。

示例：
  我明白了！先判断链表长度≥8，再判断数组长度≥64，不满足就先扩容。这个顺序我记下来了。谢谢你讲得这么清楚。

字数：≤ 70 字
```

#### 2.3 主流框架类

```
模版：
  原来{keyTakeaway}是这样的关系。你这一讲我就串联起来了！

示例：
  原来实例化→属性赋值→初始化是顺序执行的。你这一讲我就串联起来了！

字数：≤ 60 字
```

#### 2.4 中间件类

```
模版：
  我懂了！{keyTakeaway}，这样一说设计原因就很清楚了。谢谢！

示例：
  我懂了！惰性删除省 CPU、定期删除清内存，两者互补。这样一说设计原因就很清楚了。谢谢！

字数：≤ 65 字
```

#### 2.5 前端类

```
模版：
  哦原来{keyTakeaway}！我按你的例子在脑子里过了一遍，清楚了！

示例：
  哦原来 flex 更适合一维布局、grid 适合二维！我按你的例子在脑子里过了一遍，清楚了！

字数：≤ 60 字
```

### 3. 答错安慰模版

用户回答不完整或答错时，白子安慰 + 引导一起看老师讲过的关键词。

#### 3.1 数据库类

```
模版：
  没事，数据库概念多确实容易混。我们一起把老师讲的关键词找回来：{missKeywords}。

示例：
  没事，数据库概念多确实容易混。我们一起把老师讲的关键词找回来：隔离级别、脏读、不可重复读。

字数：≤ 65 字
```

#### 3.2 Java 核心类

```
模版：
  没关系，这个流程我也记了好几次。老师刚才提到的几个关键词我们再对一遍：{missKeywords}。

示例：
  没关系，HashMap 的流程我也记了好几次。老师刚才提到的几个关键词我们再对一遍：扩容阈值、树化条件、链表转红黑树。

字数：≤ 70 字
```

#### 3.3 主流框架类

```
模版：
  别急，框架步骤多一开始都这样。我们把关键节点再过一遍：{missKeywords}。

示例：
  别急，Spring 步骤多一开始都这样。我们把关键节点再过一遍：BeanPostProcessor、init-method、InitializingBean。

字数：≤ 65 字
```

#### 3.4 中间件类

```
模版：
  没关系，策略之间的区别确实不好记。我们一起看老师提到的：{missKeywords}。

示例：
  没关系，淘汰策略之间的区别确实不好记。我们一起看老师提到的：LRU、LFU、过期删除。

字数：≤ 60 字
```

#### 3.5 前端类

```
模版：
  没事，布局效果不实际写代码确实难想象。老师讲的那几个关键点我们再看看：{missKeywords}。

示例：
  没事，布局效果不实际写代码确实难想象。老师讲的那几个关键点我们再看看：BFC、浮动清除、包含块。

字数：≤ 65 字
```

### 4. 复盘陪伴模版

学习阶段结束后，白子加一句陪伴式反馈。语气因分类而异，体现这轮学习的感受。

#### 4.1 数据库类

```
我也把这些数据库概念串起来了。下次你再复习这节，我还能和你一起对一遍隔离级别。

字数：≤ 40 字
```

#### 4.2 Java 核心类

```
你讲的流程我记在笔记里了。下次看集合和多线程的题，我们再一起过。

字数：≤ 35 字
```

#### 4.3 主流框架类

```
框架的步骤我终于不晕了。以后学新的模块我还想听你讲一遍。

字数：≤ 30 字
```

#### 4.4 中间件类

```
原先我总觉得策略设计得复杂，你讲完我觉得合理多了。下次复习 Redis 再来找你！

字数：≤ 35 字
```

#### 4.5 前端类

```
你举的例子我印象很深，自己写的时候心里有底多了。

字数：≤ 25 字
```

## 协作值（Bond）绑定规则

答对或答错时，协作值变化与分类+难度挂钩：

| 分类 | 简单题答对 | 中等题答对 | 困难题答对 | 答错安慰 |
|------|-----------|-----------|-----------|---------|
| 数据库 | +1 | +2 | +3 | +0 |
| Java 核心 | +1 | +2 | +3 | +0 |
| 主流框架 | +1 | +2 | +3 | +0 |
| 中间件 | +1 | +2 | +3 | +0 |
| 前端 | +1 | +1 | +2 | +0 |

---

## Token 节省汇总

| 模版类型 | 每次调用传入字段数 | 输出字数上限 | 运行时是否调 AI |
|---------|-----------------|------------|---------------|
| 请教提问 | 3（knowledgePoint, confusedPoint, category） | ≤70字 | 否（预生成） |
| 答对反馈 | 3（keyTakeaway, bondDelta, category） | ≤70字 | 否（预生成） |
| 答错安慰 | 2（missKeywords, category） | ≤70字 | 否（预生成） |
| 复盘陪伴 | 1（category，选对应语气） | ≤40字 | 否（预生成） |

白子同桌全场景**零实时 AI 调用**，所有对话文本均为题目更新时异步预生成，缓存后直接读取。
