# 前端主页（学习流）组件与按钮总结

> 基于 `AI智能刷题重构设计与接口文档.md` 提取，面向 H5（uni-app + Vue3）前端重构。

---

## 1. 整体学习流（6个阶段）

```
选择方向 → AI老师讲课 → 课堂提问 → AI同桌互助 → 课后作业 → 学习复盘
```

对应前端 `StudyPhase` 枚举：

| Phase | 页面 | 说明 |
|-------|------|------|
| `choose` | 学习方向选择页 | 入口主页 |
| `lesson` | AI课堂页 | 老师讲课 + 课堂提问 |
| `checkpoint` | AI课堂页（提问模式） | 穿插在讲课中 |
| `classmate` | AI课堂页（同桌模式） | 课间互助 |
| `homework` | 课后作业页 | 逐题作答 + AI讲评 |
| `review` | 学习复盘页 | 成绩 + 薄弱点 + 推荐 |

## 2. 学习方向选择页（入口主页）

**对应接口：** `GET /api/study/courses?subjectId=1`

| 组件/元素 | 类型 | 说明 |
|-----------|------|------|
| 课程卡片列表 | 列表容器 | 展示 `title`, `description`, `difficulty`, `tags`, `lessonCount`, `homeworkCount` |
| 课程卡片 | 可点击卡片 | 点击后进入课程详情/创建会话 |
| 老师头像 | 图片 | 从接口返回 `teacherAvatar` |
| 同桌头像 | 图片 | 从接口返回 `classmateAvatar` |
| 难度标签 | 文本标签 | easy / medium / hard |
| 学习模式切换 | Switch / 选择器 | **快速模式**（无同桌/羁绊） vs **沉浸模式**（完整体验） |
| 开始学习按钮 | 按钮 | 点击 → `POST /api/study/sessions` 创建会话，跳转课堂页 |
| 课程描述 | 文本 | 简要说明该课程内容 |

**对应组件建议：** `CourseCard.vue`, `ModeSelector.vue`, `DifficultyBadge.vue`

## 3. AI课堂页（核心交互页）

**对应接口：** `GET /api/study/sessions/{id}/script` — 获取完整课程脚本

这页是本系统的核心，覆盖 `lesson`、`checkpoint`、`classmate` 三个阶段。

### 3.1 通用 UI

| 组件/元素 | 类型 | 说明 |
|-----------|------|------|
| AI老师面板 | 头像 + 名称 | 左上/固定位置，显示 `teacher.name`、`teacher.avatar` |
| AI同桌面板 | 头像 + 名称 | 显示 `classmate.name`、`classmate.avatar`（沉浸模式） |
| 对话气泡列表 | 列表 | 渲染 `LessonNode[]`，不同 `type` 用不同样式的气泡 |
| 学习进度条 | 进度指示器 | 当前节点位置 / 总节点数 |
| 羁绊值显示 | 数字 + 进度条 | 仅沉浸模式，`bondValue` |
| 举手提问按钮 | 浮动按钮 | 用户可随时点击提问 |

#### 已实现：SpeakerCard 多姿态 + Avatar（`frontend/Forhaed`）

课堂页角色立绘由 `SpeakerCard.vue` 渲染，姿态由 `ClassroomView.vue` 根据**节点类型 + 后端评分 `nodeResult.score`（第二层关键词评分，零 AI 调用）**自动切换。角色图通过 Vite `import` 引入（生产构建会带 hash，不能用 `/src/assets/...` 裸路径，否则 nginx 下 404）。

| 组件 | Prop | 取值 | 资产 |
|------|------|------|------|
| `SpeakerCard.vue` | `type` | `teacher` \| `baizi` | — |
| `SpeakerCard.vue` | `pose`（老师） | `idle` / `explain` / `correct` | `teacher-idle/explain/correct.png` |
| `SpeakerCard.vue` | `pose`（白子） | `idle` / `ask` / `happy` / `comfort` | `baizi-idle/ask/happy/comfort.png` |
| `Avatar.vue` | `type` / `size` | 对话框角落小头像 | `teacher-avatar.png` / `baizi-avatar.png` |

姿态切换规则（`ClassroomView.vue` computed）：

| 场景 | 老师 `teacherPose` | 白子 `baiziPose` |
|------|-------------------|------------------|
| 讲课 / 作业说明（lecture / homework_intro） | `explain` | — |
| 课堂提问 · 等待作答（checkpoint，`nodeResult===null`） | `idle` | — |
| 课堂提问 · 答对（score ≥ 70） | `explain` | — |
| 课堂提问 · 答错（score < 70） | `correct`（纠错） | — |
| 白子请教 · 首帧等待（classmate，`nodeResult===null`） | — | `idle` |
| 白子请教 · 答对（score ≥ 70） | — | `happy` |
| 白子请教 · 答错（score < 70） | — | `comfort`（安慰） |

> ⚠️ 资产缺陷：`baizi-idle.png` 当前与 `baizi-ask.png` 逐字节相同（MD5 一致），Vite 按内容哈希去重后 idle/ask 指向同一文件，视觉上无区别。代码接线已正确，等替换为真正的 idle 立绘即可自动生效。

## 7. 核心前端状态

参照文档 `StudyState` 接口：

```ts
type StudyPhase = 'choose' | 'lesson' | 'checkpoint' | 'classmate' | 'homework' | 'review'

interface StudyState {
  sessionId: number          // 当前会话ID
  courseId: number           // 课程ID
  phase: StudyPhase          // 当前阶段
  nodeIndex: number          // 当前节点索引
  homeworkIndex: number      // 当前作业索引
  currentNode?: LessonNode   // 当前节点数据
  dialogueHistory: DialogueMessage[]  // 对话历史
  userAnswer: string         // 用户当前输入的答案
  score?: number             // 当前评分
  hitKeywords: string[]      // 命中关键词
  missKeywords: string[]     // 遗漏关键词
  bondValue: number          // 羁绊值
  weakTags: string[]         // 薄弱标签
}
```

## 9. 设计约束

- 页面首先是**学习工具**，角色占页面 15%-25% 视觉权重
- 对话框像游戏，但信息密度要适合刷题
- 作业区清晰稳重，不能被角色抢走注意力
- 同桌互动每节课最多 1-2 次
- 快速模式关闭同桌和羁绊值
