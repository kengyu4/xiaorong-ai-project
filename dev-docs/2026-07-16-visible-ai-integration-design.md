# 可见 AI 课堂接入修正设计

> 日期：2026-07-16  
> 状态：方案 B 已实施并通过自动化、Docker 与 8088 代理回归  
> 范围：4.1、4.2、4.4、4.5；4.3 面试官功能暂停开发

## 一、问题与根因

1. 自由提问只出现在 checkpoint 节点且默认折叠，普通课堂提交仍展示固定评分模版，用户无法区分固定反馈与 AI 回答。
2. 当前用户即使保存了 API Key，只要没有另外保存 Provider/模型偏好，`resolveRuntimeProvider` 就返回空并静默切换 Mock。
3. Mock/降级状态未在课堂页展示，固定 Mock 文案被误认为真实 AI。
4. 深度讲评前端仅轮询约 8 秒，任务晚于轮询完成时不会再显示，也没有重试入口。
5. 个性化建议只在 session 复盘页生成，课程首页只展示薄弱标签；复盘 `nextActions` 仍是固定数组。
6. SSE 接口已经存在，但入口折叠、状态不透明，用户看不到连接、流式增量和降级状态。

## 二、批准方案

### 2.1 个人 AI 路由

- 保存个人 API Key 时同时保存当前 Provider 和模型偏好；模型为空时采用该 Provider 默认模型。
- 后端增加容错：有已配置 Secret 但没有偏好时，选择可信 Provider 中第一个已配置项及默认模型，避免静默 Mock。
- 学习 AI 响应公开安全的运行元数据：`degraded`、`providerCode`、`model`，绝不返回 API Key。
- 前端明确显示“真实 AI · Provider · 模型”或“演示/降级回复”，不得把 Mock 冒充真实 AI。

### 2.2 4.1 与 4.5 自由提问

- 所有课堂节点均显示默认展开的“问小绒老师 AI”区域。
- 支持任意文本输入；快捷问题仅作为可选项。
- 展示本节点多轮用户问题和 AI 回复，流式阶段显示连接状态和光标。
- POST SSE 继续使用 Authorization + JSON；失败时回退普通 POST。
- 普通评分反馈与 AI 自由回答使用不同标题和视觉区块。

### 2.3 4.2 低分深度讲评

- score < 70 时自动展开 AI 深度讲评。
- 轮询窗口延长至最多 60 秒，使用 1 秒间隔。
- 显示 pending/completed/failed；超时或失败时提供“重新获取”按钮。
- 显示真实 AI 或降级标识。

### 2.4 4.4 个性化建议

- 新增当前用户首页建议接口，依据全部学习记录的薄弱标签、平均分和最近课程生成 2～3 条建议。
- `/app/courses` 在薄弱点旁显示“小绒老师的学习建议”，包含加载、成功、无数据和降级状态。
- 后端按用户与学习数据指纹缓存建议，学习记录变化后重新生成，避免每次刷新消耗 Token。
- session 复盘页的 `nextActions` 改为 AI 建议解析结果，不再使用固定数组。

### 2.5 Token 上限

- free-ask：500 → 1000。
- deep-review：400 → 800。
- personal-review：350 → 700。
- interview-follow-up：300 → 600；仅调整上限，不继续开发 4.3 页面和交互。
- 自由提问 Prompt 的字数约束同步由 300 字调整为 600 字。

## 三、安全边界

- 浏览器只提交 API Key 一次；保存后立即清空，不回显明文。
- API Key 继续使用服务端 AES-GCM 加密存储。
- AI 回答仅返回 Provider、模型和降级状态，不返回 Secret、Base URL 鉴权头或上游原始错误。
- 所有学习数据、建议缓存、会话和异步任务均按当前用户隔离。

## 四、验收标准

1. 仅保存 API Key 后，无需再点一次“保存模型”，后续学习请求即可选择真实 Provider。
2. 课堂任意节点都能输入自由问题并在页面看到增量 AI 回复。
3. 页面明确区分真实 AI 与 Mock/降级回复。
4. 低分后 60 秒内完成的深度讲评可自动显示；超时可手动重新获取。
5. `/app/courses` 可看到薄弱点和 2～3 条个性化建议。
6. Token 上限分别为 1000、800、700、600。
7. 4.3 面试官页面及业务交互不在本次实现范围。
8. 后端测试、前端测试、类型检查、生产构建和 Docker 真实链路全部通过。
## 五、实施结果（2026-07-16）

- 个人 Secret 保存与 Provider/模型偏好同步完成；后端已增加“Secret 存在、偏好缺失”的可信 Provider/default model 容错。
- 课堂所有节点均可见自由提问和按节点多轮历史；SSE `delta/done`、连接状态和普通 POST 回退已接入。
- 低分讲评使用最长 60 秒轮询和重试；首页/复盘均展示当前用户个性化建议。
- 所有学习 AI 内容展示安全的 `providerCode`、`model`、`degraded`，不返回 Secret。
- Token 上限为 free-ask 1000、deep-review 800、personal-review 700、interview-follow-up 600。
- 4.3 面试官页面和完整业务维持暂缓，不计入本次完成范围。
- 最新 Docker 镜像启动正常；`http://localhost:8088/` 返回 200，未登录 runtime/courses API 返回 401，不再出现 502。
