# AI 角色设定与生图提示词

## 1. 设计目标

角色服务于学习，不做强恋爱养成。

目标：

- 提高学习积极性。
- 让 AI 老师和 AI 同桌有记忆点。
- 画风统一，放到刷题页面里不突兀。
- 减少明显 AI 生成感。
- 避免油腻、暧昧、过度写实、过度二次元。

推荐定位：

```text
AI 老师：温和、专业、讲得清楚。
AI 同桌：可爱、认真、会向用户请教问题。
AI 面试官：克制、专业，后续用于模拟追问。
```

## 2. 参考图转化原则

你给的参考图可以提炼这些方向：

- 小巧圆润。
- 灰白低饱和配色。
- 表情有一点委屈或认真。
- 有轻微猫耳 / 软帽轮廓记忆点。
- 重点是可爱和陪伴，不是性感。

不要直接照抄：

- 不要视频截图感。
- 不要字幕、水印、手机状态栏、播放按钮。
- 不要真实拍摄风格。
- 不要过度高光塑料感。

更适合本项目的方向：

```text
从短视频 AI 萌图，转成教育产品里的轻量 2.5D 应用插画角色。
```

## 3. 统一画风

推荐英文风格词：

```text
soft 2.5D app illustration, matte texture, low saturation, rounded shape, clean silhouette, subtle hand-drawn linework, simple readable expression, soft ambient light, transparent background, mobile learning app UI friendly
```

推荐中文风格词：

```text

```


## 4. 角色一：AI 老师

名称：

```text
小绒老师
```

定位：

```text
温和专业的 AI 老师，像认真备课的年轻助教，负责讲课、提问、纠错、作业讲评。
```

性格：

- 讲解有条理。
- 先鼓励，再纠错。
- 不说废话。
- 用户答错时换一种更简单的讲法。

外观：

- 短发或中短发。
- 简洁校园风开衫。
- 手里拿平板或讲义。
- 蓝灰、白、浅青绿配色。
- 不做强性别吸引。

通用 prompt：

```text
原创 AI 老师角色，年轻助教气质，温和专业，短发，简洁校园风开衫，手里拿着平板讲义，蓝灰和白色低饱和配色，柔和 2.5D 应用插画风格，哑光质感，圆润轮廓，干净剪影，轻微手绘线条，表情清楚自然，移动端学习应用 UI 角色，透明背景，柔和环境光，不要复杂背景。
```

讲解动作：

```text
小绒老师正在温和讲解知识点，一只手轻轻指向旁边的透明讲义板，表情认真但不严肃，适合课堂对话框旁边展示。
```

纠错动作：

```text
小绒老师表情温和，手指轻点讲义，像在提醒一个容易混淆的知识点，没有责备感，透明背景。
```

## 5. 角色二：AI 同桌

名称：

```text
白子同桌
```

定位：

```text
坐在旁边一起学习的 AI 同桌。她会向用户请教问题，也会在用户答错时一起复盘。重点是陪伴和互助，不是恋爱。
```

性格：

- 有一点点容易困惑。
- 会认真听用户解释。
- 答对时真诚感谢。
- 答错时不打击用户。
- 常说“我们一起再问问老师吧”。

外观：

- 小巧圆润。
- 灰白或暖白发色。
- 猫耳形发饰或柔软帽子轮廓。
- 浅灰围巾或宽松校园外套。
- 大眼睛但不要过度闪亮。
- 不露骨，不暧昧，不恋爱化。

通用 prompt：

```text
原创 AI 学习同桌角色，小巧圆润，灰白低饱和配色，暖白短发，猫耳形发饰或柔软帽子轮廓，围着浅灰围巾，宽松校园外套，大而清楚的眼睛，表情认真又有一点点困惑，像正在向用户请教问题，柔和 2.5D 应用插画风格，哑光质感，干净剪影，轻微手绘线条，移动端学习应用 UI 角色，透明背景，亲切但不暧昧，不性感。
```

请教动作：

```text
白子同桌双手抱着笔记本，微微抬头看向用户，表情认真又有一点困惑，像在说“这个知识点你能讲给我听吗”，透明背景。
```

答对开心：

```text
白子同桌露出真诚开心的表情，轻轻点头，像在感谢用户讲明白了问题，动作自然，不夸张，透明背景。
```

答错安慰：

```text
白子同桌轻轻摆手，表情温和，像在说“没关系，我们一起再看一遍”，不沮丧，不夸张哭泣，透明背景。
```

## 6. 角色三：AI 面试官


## 10. 生成参数建议

Midjourney：

```text
立绘：--ar 3:4 --v 6 --style raw --s 120
头像：--ar 1:1 --v 6 --style raw --s 100
封面：--ar 16:9 --v 6 --style raw --s 80
```

Stable Diffusion / Flux：

```text
size: 1024 x 1024
steps: 24-32
cfg: 4.5-7
sampler: DPM++ 2M Karras 或模型推荐采样器
seed: 同一角色尽量固定 seed 或固定参考图
```

减少 AI 感：

- 少用 `masterpiece`、`ultra detailed`、`8k`、`cinematic`。
- 多用 `matte texture`、`low saturation`、`clean silhouette`。
- 输出后统一裁切、压缩、调低对比度。
- 页面里用轻阴影和低饱和背景承接角色。

## 11. 页面使用建议

AI 老师：

- 放在课堂页左侧或对话框旁。
- 用于讲课、纠错、作业讲评。
- 不遮挡题目和输入框。

AI 同桌：

- 只在课间互助、答对、答错时出现。
- 适合右下角小立绘。
- 每节课最多出现 1 到 2 次，避免抢戏。

羁绊值命名建议：

```text
协作值
一起复盘
学习伙伴
讲清楚了 +3
```

不建议：

```text
好感度暴涨
心动值增加
亲密度爆表
```

## 12. 角色台词基调

AI 老师答对：

```text
这个回答方向是对的。你已经抓住了核心点，我帮你补一句更适合面试的表达。
```

AI 老师答错：

```text
这里容易混淆。我们先退一步，只看这个概念最关键的边界。
```

AI 同桌请教：

```text
我刚才有点没听懂，你能用自己的话给我讲一下吗？
```

AI 同桌答对：

```text
谢谢你，这样一讲我就清楚多了。原来重点在这个边界上。
```

AI 同桌答错：

```text
没关系，我也卡在这里。我们一起把老师刚才说的关键词找回来。
```

第一版只做两个角色：

```text
白子同桌：负责请教、鼓励、一起复盘。
```

```text
低饱和、圆润、哑光、轻 2.5D、透明背景、教育产品 UI 友好。
```

产品关键词：

