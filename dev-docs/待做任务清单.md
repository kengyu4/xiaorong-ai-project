### 完成说明（2026-07-16）

- 修复“前端已保存个人 API Key 但学习请求仍进入 Mock”：保存 Secret 时同步保存当前 Provider/模型偏好；后端在 Secret 存在但偏好缺失时，按可信 Provider 优先级选择已配置 Provider 与默认模型。
- `/admin/runtime` 支持百炼、DeepSeek、OneAPI 等后端可信 Provider，支持上游模型列表和手工模型名切换；Secret 保存后立即清空输入框，查询仅返回末四位脱敏信息。
- 学习 AI 响应只公开 `providerCode`、`model`、`degraded`，前端明确展示“真实 AI”或“演示/降级回复”，不返回 API Key、密文、IV、Authorization 或上游原始错误。
- 课堂所有节点均默认显示“问小绒老师 AI”，按节点保留多轮对话；POST SSE 展示连接/生成/完成/失败状态，并在传输失败、缺少 `done` 或无正文时自动回退普通请求。
- 低分深度讲评轮询延长到 60 秒并提供重试；`/app/courses` 展示当前用户真实薄弱点以及 2-3 条个性化建议；复盘 `nextActions` 使用 AI 建议解析结果。
- 新增 `/api/study/overview/advice`，建议按用户和学习数据指纹缓存；`free_ask` 记录不参与平均分，避免自由提问污染学习成绩。
- 4.3 按用户要求暂缓：本次不声明面试官页面或完整业务已完成，只保留预算翻倍和已有后端基础。

### 验证证据（2026-07-16）

- 后端：`mvn -q test`、`mvn -q -DskipTests package`。
- 前端：`npm test`（11/11）、`npm run type-check`、`npm run build`。
- Docker：使用最新代码重建 `app`、`frontend`；MySQL、Redis、RabbitMQ 均为 healthy，应用正常启动，无 Nacos/`127.0.0.1:9848`/启动失败错误。
- 8088 回归：`GET /` 返回 200；未登录访问 `/api/admin/runtime/status` 与 `/api/study/courses?subjectId=1` 返回预期 401，而不是 502。
- 安全检查：修改过的前后端源码无 UTF-8 BOM、无 连续问号乱码；浏览器无明文 Key 持久化，接口运行元数据不包含 Secret。
### 4.5 百炼原生 SSE 修正（2026-07-16）

- 百炼调用不再使用 `java.net.http.HttpClient`，改为 Spring AI Alibaba `DashScopeChatModel.stream(Prompt)`；POST SSE 端点使用非阻塞 `subscribe()`，避免生成完成后才一次性返回。
- 前端对上游返回的大 `delta` 块增加逐字消费队列，以 16ms/字渲染；`done` 必须等字符队列清空后才完成，流失败时取消未渲染字符。
- 2026-07-16 修复逐字渲染的 Vue 响应式根因：流式助手消息改为 `reactive<AskMessage>` 代理后再写入历史数组，后续每个字符均修改响应式代理并触发 DOM 重绘，避免修改原始对象导致内容一直积累到结束状态才整段显示。
- 前后端统一 `ready` / `delta` / `done` 事件：`ready` 立即建立可见连接状态，`delta` 逐块追加文本，`done` 携带 Provider、模型和降级元数据；Nginx 与 Controller 均禁止缓冲/缓存。
- 自由对话 System Prompt 固定为“小绒老师”身份，要求结论先行、3–5 句、短句、禁止 Markdown 星号强调；后端与前端双重清理 AI 回答中的 `**粗体**` 标记。
- `/admin/runtime` 保留 DeepSeek、百炼、OneAPI 切换以及任意模型名手工输入；个人 Key 仍仅在服务端 AES-GCM 加密保存，浏览器只看到 `****末四位` 脱敏值，不会因去除 AI 回答星号而破坏 Key 的安全边界。
- 百炼适配器错误信息已改为可读且不包含上游响应或 Key 的安全文案，不再出现 `????` 占位符。
- 前端生产构建改为顺序执行 `type-check` 后再执行 `build-only`，消除 Windows + Vite 8 并行构建时偶发的 `index.html` 绝对路径竞争。
- 验证覆盖 adapter、SSE service/controller、前端解析与状态更新、星号清理和安全错误文案；后端全量 Maven 测试/打包、前端 12/12 测试/生产构建、Docker app/frontend 与 8088 代理回归均纳入验收。
