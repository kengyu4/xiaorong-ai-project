## 第二部分：对话固定模版

### 2.1 模版体系总览

整个对话系统分为三层：

```
第一层：固定模版（预生成，零 AI 调用）
  ├── 小绒老师：知识点讲解、课堂提问、作答评估、纠错
  ├── 白子同桌：课间请教、答对反馈、答错安慰、复盘陪伴
  └── 岚川面试官：面试开场、追问、评分、复盘

第二层：规则评分（关键词匹配，零 AI 调用）
  └── 用户答案命中/遗漏关键词判定

第三层：自由对话（实时 AI 调用）
  ├── 用户举手自由提问
  ├── 低置信度深度讲评
  ├── 面试官多层追问
  └── 个性化复盘
```

### 2.4 岚川面试官固定模版（新增）

#### 设计原则

```
- 面试官的话术要比老师更短、更克制
- 不鼓励、不打击、不展开讲解
- 追问必须基于用户刚答的内容，不能自说自话
- 全程控制在 "提问 → 倾听 → 追问 → 评分" 的节奏
```

#### ① 面试开场白

```
常量名：INTERVIEWER_OPENING_TEMPLATE

场景：用户进入模拟面试模式时触发

固定前缀（随机选一句）：
  A: 接下来我会问你几道{subjectName}方向的题，每道题我会根据你的回答追问。准备好了就开始。
  B: 模拟面试现在开始。覆盖{subjectName}方向，共{questionCount}道题。直接回答即可。
  C: 不用紧张，就当是一次练习。我会从{subjectName}开始提问。

字数：≤ 50 字
```

#### ② 面试提问

```
常量名：INTERVIEWER_QUESTION_TEMPLATE

场景：面试官出题

固定前缀：
  第{index}题：
模版：
  {topicName}
  提示：{keyHint}
  请开始你的回答。

字数：≤ 40 字
```

#### ③ 追问

```
常量名：INTERVIEWER_FOLLOW_UP_TEMPLATE

场景：用户回答后，面试官追问

规则：
  第一层追问：从"是什么"到"为什么"
    话术：你说到了{userPoint}，能进一步说说它为什么是这样吗？
  第二层追问：从"为什么"到"如果变化"
    话术：如果{scenarioChange}，你的结论还成立吗？
  第三层追问：从"理论"到"实战"
    话术：你在实际项目中遇到过{relatedScenario}吗？怎么处理的？

每层：≤ 50 字
最大追问层数：2 层（超过自动结束本题）
```

#### ④ 面试评分

```
常量名：INTERVIEWER_SCORE_TEMPLATE

场景：追问结束后，面试官给本题打分

输出 JSON：
{
  "score": 0-100,
  "level": "优秀|良好|一般|需加强",
  "hitKeywords": [],
  "missKeywords": [],
  "interviewerComment": "（克制、专业，一句话总结）",
  "risk": "（面试中可能被继续追问的点）"
}

评分权重：
  - 核心概念准确：40%
  - 表达清晰度：30%
  - 深度与扩展：30%
```

#### ⑤ 面试复盘

```
常量名：INTERVIEWER_REVIEW_TEMPLATE

场景：所有题目结束后，生成面试报告

输出 JSON：
{
  "overallScore": 0-100,
  "level": "优秀|良好|一般|需加强",
  "strengthTags": [],
  "weakTags": [],
  "summary": "一句话总结",
  "riskPoints": [],
  "suggestions": [
    "建议 1",
    "建议 2",
    "建议 3"
  ],
  "recommendedCourses": []
}
```

字数：≤ 200 字

## 第三部分：自由对话规划

### 3.1 自由对话 vs 固定模版的边界

| 场景 | 走固定模版 | 走自由对话（AI） | 说明 |
|------|-----------|----------------|------|
| 老师讲课 | ✅ 全部预生成 | ❌ | 题库固定，预生成后缓存 |
| 老师课堂提问 | ✅ 全部预生成 | ❌ | 同上 |
| 老师作答评估（高置信度） | ✅ 关键词评分 | ❌ | score ≥ 70 分时不调 AI |
| 老师作答评估（低置信度） | ❌ | ✅ score < 70 | 启用深度讲评 |
| 老师纠错 | ✅ 固定模版 | ❌ | 按分类走固定话术 |
| 同桌课间请教 | ✅ 全部预生成 | ❌ | 题库固定 |
| 同桌答对/答错反馈 | ✅ 固定模版 | ❌ | 零 AI 调用 |
| **用户举手自由提问** | ❌ | **✅ 实时 AI** | 唯一必须走 AI 的核心场景 |
| **面试官追问（用户回答在预期内）** | ✅ 模版追问 | ❌ | 关键词匹配 |
| **面试官追问（用户回答出人意料）** | ❌ | **✅ 实时 AI** | 需要理解语义 |
| **面试评分** | ✅ 关键词 + JSON | ❌ | 固定结构 |
| **面试报告** | ✅ JSON 模版 | ❌ | 固定结构 |
| **个性化学习建议** | ❌ | **✅ 实时 AI** | 基于学习记录动态生成 |

### 3.2 自由对话场景详细设计

#### 场景 A：用户举手自由提问

```
触发方式：所有课堂节点默认展示“问小绒老师 AI”区域 → 选择快捷问题或自定义输入

快捷问题（预置 2-3 个，与当前知识点相关）：
  - 示例：当前讲 ref/reactive → 快捷问题 ["ref 在模板里为什么不用 .value？", "reactive 解构后怎么保持响应式？"]

请求参数：
  - sessionId, nodeId, question（用户输入的文字）
  - 当前节点上下文由服务端读取，浏览器不能提交任意历史、Provider、模型或 Token 上限

AI 约束：
  系统提示词：
    你是{currentRole}，只能围绕当前课程知识点{currentKnowledgePoint}回答。
    要求：
    1. 不超过 600 字。
    2. 不展开无关知识点。
    3. 如果问题超出范围，先简单回答，再引导回当前课程。

上下文长度限制：
  - 最多传当前节点上下文 + 最近 3 轮对话
  - 不传整套题库、不传用户资料

响应方式：
  - 普通模式：POST 请求，完整返回
  - 长回答模式：SSE 流式返回 event: delta

Token 预算：
  - 每次自由提问 ≤ 1000 token（含上下文）
```

#### 场景 B：低置信度深度讲评

```
触发条件：用户提交答案 → 关键词评分 score < 70

流程：
  1. 先返回关键词评分结果（速度优先）
  2. 同时异步调用 AI 生成深度讲评
  3. 前端轮询或 SSE 推送讲评结果

AI 约束：
  系统提示词：
    你是小绒老师。用户提交的答案不够完整，请做深度讲评。
    要求：
    1. 先肯定正确的部分。
    2. 指出遗漏的关键点和原因。
    3. 给一个更标准的表达示例。
    4. 不超过 250 字。

上下文：
  - 题目 + 标准答案摘要 + 用户答案 + 已命中/遗漏关键词
  - 不加历史对话

Token 预算：
  - 每次 ≤ 800 token
```

#### 场景 C：面试官自由追问

```
触发条件：用户回答超出预期关键词范围，需要理解语义后追问

判断规则：
  - 用户答案长度 > 80 字 → 走自由追问
  - 用户答案包含未预期关键词 → 走自由追问
  - 用户答案太短（< 15 字）→ 走固定追问模版

AI 约束：
  系统提示词：
    你是岚川面试官。基于用户刚刚的回答，提出一个追问。
    要求：
    1. 只问一个问题。
    2. 追问必须基于用户答案中的具体内容。
    3. 追问要深入一层（从"是什么"到"为什么"或"怎么做"）。
    4. 不超过 50 字。
    5. 如果用户已答到足够深度，输出 "NO_FOLLOW_UP"。

上下文：
  - 题目 + 用户答案 + 追问层级（仅传当前这题，不传之前题目）
  - 如果用户已答到足够深度，不追问，自动进入评分

Token 预算：
  - 每次 ≤ 600 token（当前仅保留预算与后端基础，面试官页面和完整业务暂缓）
  - 最多 2 层追问 → 理论最多 1200 token/题；当前面试官功能暂缓，不纳入本次完成验收
```

#### 场景 D：个性化复盘建议

```
触发条件：学习阶段/面试结束后生成个性化建议

流程：
  1. 先组装固定复盘数据（平均分、错题、薄弱点）
  2. AI 根据用户的具体薄弱点生成 2-3 条针对性建议
  3. 固定数据 + AI 建议合并输出

AI 约束：
  系统提示词：
    基于用户的学习数据，生成个性化学习建议。
    要求：
    1. 针对薄弱点{weakTags}提出具体可执行的建议。
    2. 每条建议包含"做什么"和"为什么"。
    3. 不超过 150 字。

上下文：
  - 薄弱标签 + 平均分 + 最近学习的课程名
  - 不加用户历史数据

Token 预算：
  - 每次 ≤ 700 token
```

### 3.3 Token 总预算与成本控制

| 场景 | 每次消耗 | 频率 | 日均估算（1000 用户） |
|------|---------|------|---------------------|
| 固定模版 | 0 token | 每次交互 | 0 |
| 举手自由提问 | ≤1000 token | 每节课 0-2 次 | 按实际用量监控 |
| 低置信度讲评 | ≤800 token | 约 20% 的答题 | 按实际用量监控 |
| 面试自由追问（暂缓） | ≤600 token | 每题最多 2 次 | 暂不计入当前上线用量 |
| 个性化复盘 | ≤700 token | 学习数据变化时生成并缓存 | 按实际用量监控 |
| **合计** | | | **以 `/api/study/ai/budget` 实际统计为准** |

> 2026-07-16 实施状态：4.1、4.2、4.4、4.5、4.6 已完成；4.3 面试官完整页面与业务流程按需求暂缓。所有真实 AI 响应只展示安全的 Provider/模型/降级状态，绝不回传 API Key。
### 3.4 自由对话的降级策略

| 场景 | 降级方案 |
|------|---------|
| AI 超时/不可用 | 返回预设兜底话术："我暂时无法回答这个问题，建议你先看看相关知识点。" |
| 自由提问内容不合适 | 敏感词过滤 → "这个问题暂时超出了学习范围，我们回到当前知识点继续吧。" |
| 低置信度讲评 AI 失败 | 返回关键词遗漏列表 + 标准答案（不加 AI 分析） |
| 面试追问 AI 失败 | 使用固定追问模版兜底 |
| 复盘 AI 失败 | 只返回统计数据，不加个性化建议 |
