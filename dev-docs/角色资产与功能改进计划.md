## 3. 岚川面试官 — 功能规划

### 3.1 功能定位

面试官不应进入普通刷题流程，而是作为独立的**模拟面试模式**，在用户完成一定量的学习后开启。

```
普通刷题（小绒老师 + 白子同桌）
    ↓ 掌握度达标后解锁
模拟面试（岚川面试官）
    ↓
面试结束 → 生成面试报告 → 推荐复习方向
```

### 3.2 面试流程设计（TODO）

```
面试官开场（介绍面试规则和专题范围）
    ↓
提问第 1 题（从已学专题中选题）
    ↓
用户回答
    ↓
面试官追问（1~2 层，基于用户回答动态生成）
    ↓
面试官评分（正确率 + 表达清晰度 + 深度）
    ↓
提问第 2 题（循环）
    ↓ ...（3~5 题一轮）
    ↓
面试结束
    ↓
生成面试报告：得分、薄弱点、改进建议、推荐复习专题
```

#### 3.3.4 新增数据库表

```sql
create table ai_interview_session (
    id bigint primary key auto_increment,
    user_id bigint not null,
    status varchar(20) default 'in_progress',  -- in_progress / finished
    subject_ids json,          -- 本次面试覆盖的专题
    total_score int,           -- 总分
    weak_tags json,            -- 薄弱标签
    question_count int,        -- 总题数
    started_at datetime,
    finished_at datetime,
    create_time datetime
);

create table ai_interview_record (
    id bigint primary key auto_increment,
    session_id bigint not null,
    topic_id bigint,
    question_text text,          -- 面试官提出的问题
    user_answer text,            -- 用户回答
    follow_up_question text,     -- 追问问题
    follow_up_answer text,       -- 追问的回答
    score int,                   -- 本题评分
    hit_keywords json,
    miss_keywords json,
    interviewer_comment text,    -- 面试官评语
    risk text,                   -- 面试风险点
    create_time datetime
);
```

### 3.4 解锁条件设计（TODO）

面试模式不应一开始就开放，需要设定解锁条件：

| 条件 | 说明 |
|------|------|
| 完成至少 2 个专题的学习 | 确保用户有足够知识储备 |
| 专题掌握度 ≥ 60% | 低于此值建议继续学习而非面试 |
| 可选的：每日首次免费，后续消耗"面试券" | 控制频次，后续迭代考虑 |
